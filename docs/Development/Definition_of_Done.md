# Definition of Done

A feature is complete only if all of the following are true:

- [ ] Business rules implemented, matching `docs/Product/Business_Rules.md`.
- [ ] Documentation updated (relevant `docs/` files, `CHANGELOG.md`, and `PROJECT_MANIFEST.md`/`CLAUDE.md` if project-wide rules changed).
- [ ] Validation completed (automated checks run and passing, behavior manually verified where it has a runtime surface).
- [ ] No duplicated code.
- [ ] No lint errors.
- [ ] No type errors.
- [ ] Tests created, when applicable.
- [ ] Committed as one logical commit with a Conventional Commits message.
- [ ] Pushed to the remote repository.

A feature missing any of the above is not done and must not be left committed in that state.
